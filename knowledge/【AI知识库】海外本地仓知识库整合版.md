# 一、退件服务概况

## 1.1 服务项目流程

基于菜鸟提供的数据，X公司提供以下服务给菜鸟：

1.  **收货入库**
    
    *   免质检销毁
        
    *   质检入库
        
2.  **质检**：实物和系统信息对比，执行质检并提交质检结果。（要求退回仓库的包裹，需要在到仓后2个工作日内完成质检）
    
3.  **装箱**
    
    *   **打包（小包）**：根据系统生成的出库策略，进行对应SKU打包
        
        *   本地销毁：对应SKU直接进行销毁
            
        *   退国内/退香港：根据系统生成订单LP单，将对应SKU进行打包操作，完成贴面单
            
        *   本地自提：根据系统生成出库单，将对应SKU进行打包操作，完成贴面单，待商家自提
            
    *   **组包（大包）**：
        
        *   关联大包：将小包对应流向一致的大包，进行逐票扫描，系统绑定
            
        *   存储大包：将不同流向的大包/大托，做好分区管理，等待出库
            
4.  **出库**：xx天内根据干线提货时间完成出库，仓点击出库成功
    
5.  **交航**：仓点击交航成功

### 2.3.2 收货入库详细作业流程

退货包裹由快递公司承运送到仓库，作业人员使用 **RMS** 系统对快递单号进行扫码录入操作。扫描录入后按不同策略（质检、免质检、取消、无主件）进行分堆处理，送至对应的操作台：

*   **质检（QC）**：需质检拆包的送到质检台
    
*   **免质检（destroy）**：放置销毁件区域
    
*   **取消（cancel）**：订单为取消状态放置取消件区域
    
*   **无主件（abnormal）**：放置异常件区域
    

_注：AE 退货包裹包含 LP开头单号、消费者自寄包裹_

**包裹查验：** 收货时进行包裹正反面查验（如遇箱体，按6面查验）。验视包裹面单完整性：有无面单、一个包裹是否含多个不同面单、是否是重复使用的面单，如有上述异常进入异常件处理流程。

**系统操作说明（退货收包扫描）：** 打开RMS APP收货(Inbound)模块 → 扫描包裹物流单号（仅支持国际运单号，勿用LP条码扫） → 扫描成功后系统显示分堆结果：Q(质检)、C(取消)、D(销毁)、A(异常/无主件)，按结果将包裹分堆放置。

**退货收包扫描系统界面：**

**异常处理：**

*   ①**扫描重复**(Already Inbounded)：在APP"操作历史列表"查询核对，如一致则跳过，如不一致则存入异常件区域并登记为重复面单
    
*   ②**扫描失败**(Scan Failed)：对准面单条码重试，仍无效则说明系统无匹配退货单，需在PC端操作无主件异常登记，包裹单独存放
    
*   ③**无主件**(ABNORMAL)：用运单号扫描入库，点击"确认(Confirm)"跳过，实物存放异常件区域
    
*   ④**取消件**(CANCELLED)：包裹物流单已被取消，单独存放

## 2.4 质检（Inspection）

**支持端：只支持APP**

### 2.4.1 质检流程

**第一步：扫描面单**

*   点击质检(Inspection)进入
    
*   扫描包裹物流单号
    
*   系统带出待质检的货品明细
    

**第二步：拍照**

*   完成外包装照片（保证面单号完整清晰可见）
    
*   所有商品照片
    
*   单个商品拍照点击中心圆圈
    
*   完成后点击"NEXT"
    

**第三步：质检判定**

*   逐项核实货品品质和数量
    
*   首先判定为正品（点击"正品GOOD"）
    
*   若发现缺失、系统策略错误等，可点"cancel"后重新质检
    

**第四步：确认提交**

*   检查该包裹所有SKU已完成质检
    
*   品类属性正确，确认质检信息无误
    
*   点击完成
    

### 2.4.2 质检策略说明

确认质检结果后系统展示出库策略（按SKU维度），共计**7种：**

*   **Self-Pickup**：本地自提
    
*   **CN**：退国内
    
*   **HK**：退香港
    
*   **DESTROY NOW**：本地销毁
    
*   **Destroy After**：延期销毁
    
*   **质检后二售**
    
*   **清货计划-自提**
    

**注意：** 商品良残判定涉及到出库策略变化，请结合凭据谨慎判断。

### 2.4.3 已质检列表

如果你想要重新打印面单或者想要检查已质检的情况，点击扫描页面右上角列表，进入已质检列表。

**注意：** 进入质检详情，请确保点击的是单据卡片的空白信息，因为很容易点到LP和运单号。

### 2.4.4 特殊流程

1.  **质检sku数量 > 预报sku数量（黑标商品）：** 如果这个sku不属于这个包裹，你可以在左侧点击➕号增加sku，系统会自动将它标记为黑码
    
2.  **质检件数 < 预报件数：** 在质检原因中选择"丢失"，并在此理由下填入相应的丢失数量
    
3.  **质检件数 > 预报件数：** 选择对应的质检原因，并在对应理由下填入相应的数量
    
4.  **修改质检结果：** 质检结果未提交前，可返回修改；质检结果提交后，不支持修改
    
5.  **打印面单：** 你需要前往PC端进行面单打印 → 建议一个入库单质检完成后即打印对应出库单面单，否则面单与包裹难匹配准确
    

### 2.4.5 质检异常处理

**少货（lost）：**

*   场景描述：如果包装盒内商品已经全部扫描/核对完毕，但是系统显示包裹中仍旧有剩余商品未扫描完
    
*   处理方式：再次仔细检查包裹内是否有遗留商品没有处理，如有，继续处理直至商品全部被处理完。如果确实没有商品遗留在箱内，在系统上"丢失（lost）"点击"+"，增加到对应缺失的数量（和预报做对比），完成后点击"提交submit"，并继续质检下一个包裹
    

**破损（damage）：**

*   场景描述：存在商品出现破损（破洞、明显完全无法使用）情况
    
*   处理方式：在系统上"破损（DAMAGE）"点击"+"，增加到对应的数量，并对破损部位加拍1~2张照片，完成后继续质检下一个SKU
    

**货不对板（APPEARANCE MISMATCH）：**

*   场景描述：实际商品与计划退回商品外观不相符的情况，如品类不符、款式不符、颜色不符、明显大小不符
    
*   处理方式：在系统上"货不对板（APPEARANCE MISMATCH）"点击"+"，增加到对应的数量，完成后继续质检下一个SKU
    

**漏液（LEAK LIQUID）：**

*   场景描述：存在瓶装、罐装或其他容器装漏液的情况
    
*   处理方式：在系统上"漏液（LEAK LIQUID）"点击"+"，增加到对应的数量，并对漏液部位加拍1~2张照片，完成后质检下一个SKU
    

**零件打开【MULTIPLE PARTS 50+】& UNPACKED：**

*   场景描述：涉及多零件组装类商品，内物零件超过50件，商品是否有打开的情况
    
*   处理方式：在系统上"零件打开【MULTIPLE PARTS 50+】& UNPACKED"点击"+"，增加到对应的数量，对该SKU加拍1张照片，完成后继续质检下一个SKU
    

**粉膏类内物打开【POWDER/LIQUID】& UNPACKED：**

*   场景描述：粉膏液类目商品，内物商品有打开的情况
    
*   处理方式：在系统上"【POWDER/LIQUID】& UNPACKED"点击"+"，增加到对应的数量，并对打开痕迹加拍1~2张照片，完成后质检下一个SKU
    

**商品变形【SHAPE CHANGED】：**

*   场景描述：存在商品发生形状改变的情况，商品形状改变且无法复原
    
*   处理方式：在系统上"【SHAPE CHANGED】"点击"+"，增加到对应的数量，并对变形痕迹加拍1~2张照片，完成后质检下一个SKU
    

**多件：**

*   如果系统显示包裹内剩余商品数量0，但是包裹内仍旧有商品剩余，对比预报SKU，如是系统预报的商品，则按质检结果（良/残）后将商品放入对应框。如果不是系统无法识别商品/多于预报数量的商品，请参考黑码处理流程
    

**黑码：**

*   如果系统无法识别商品，在系统上左侧SKU预报图片处点击"+"，补充商品照片，增加到对应的数量，完成后点击"提交submit"。对应商品按销毁处理
    

## 2.5 小包打包

### 2.5.1 小包打包作业流程

根据不同的装箱策略和带电标识，将对应SKU及数量打为小包，（完成称重）完成包裹封口，贴上对应的面单。

**打包的方式：**

*   将需"退国内/退香港/本地自提/本地销毁**/质检后二售/清货计划—自提**"SKU单品做复核，装入PE袋/箱子打包为小包，粘贴"退国内/退香港/本地自提/本地销毁**/质检后二售/清货计划—自提**"LP面单
    
*   出库面单覆盖入库面单
    

**注意：**

1.  黑码商品合并到本地销毁策略中，完成销毁即可
    
2.  同一个策略下，带电和不带电的包裹分别打包
    

### 2.5.2 小包打包系统操作

1.  **打包：** 整理SKU及对应数量，出库策略显示为"Self-Pickup"(本地自提)、"CN"（退国内）、"HK"（退香港）、"DESTORY NOW"（本地销毁）、"Destroy After 'yyyy-mm-dd'"（在'yyyy-mm-dd'日期销毁）**、质检后二售、清货计划—自提**，共计**7种**，按左侧展示该策略下合计的SKU种类数和总物品数量，图片右下角展示该SKU应包含的数量，需核实清晰，准确装入对应的商品和数量
    
2.  **称重（可选）：** 装入完成后，可以进行小包裹称重（直接由硬件设备采集结果），不支持直接输入重量数据
    
3.  **打印面单：** 完成内物核对后，点击打印运单(Print Waybill)，将包裹进行封口，将面单进行粘贴。（支持批量打印/重复打印，但需注意准确粘贴，包裹打包完成后产生的异常不可逆）
    

## 2.6 大包装箱（组包）

**支持端：只支持APP**

### 2.6.1 组包作业流程

1.  **大包绑定：** 将小包对应出库策略一致的大包，进行逐票扫描，系统绑定。逐票扫描，装大包（系统扫描完成，即为大小包映射关系绑定）
    
    *   **注意：** 不同流向小包不能混装；普电不能混装
        
2.  **自提线上化组包操作：** 仓库人员定期查看系统中预约待组包的提醒，操作流程如下：
    
    *   组包扫描小包条码，如果商家已预约则会显示预约标识，反之则无预约标识
        
    *   已预约的小包组同一个大包，未预约的小包组同一个大包，二者不可也不支持混包
        
    *   同一个商家，不同预约日期的自提小包，不支持组同一个大包
        
    *   组包过程中如遇某个小包因丢失/破损等原因无法组包，需删除对应小包后按照预约单继续组大包
        
    *   "已预约"组包过程中，如果商家取消，则整个预约单被取消，阻断当前组包操作
        
    *   "已预约"组包完成后，在PC端点击预约单打印、发货资料打印（服务商自提场景下），并把面单粘贴在大包上，预约单等自提交接时签字使用
        
3.  **大包存储：** 依据业务要求，补充标识（如要求口岸清晰（退国内/退香港）、数量标记等），将自提/在库等待货物做好库内大包存储
    
    *   等待至商家预约时间进行提货；如果商家一直未提货，等待系统销毁指令
        
    *   退国内/退香港存储到交干日期前
        
    *   全托商品需要以大包形式在库存储26天（美国12天），存储到期后按大包完成销毁
        

**注意：** 部分仓有消退&不可达两种业务，请将消退的包裹和不可达的包裹分开提供清单，实物包裹也分开打包，箱子上清晰明显标注消退和不可达标签。短期不可达不适用RMS系统。

### 2.6.2 组包系统操作

**作业流程：** 大包装箱时，需在小包封口后粘贴唯一面的大包面单。面单需覆盖原入库面单，避免出库运输时扫描到入库面单信息。同一出库策略下，带电与不带电SKU须分开打包。普通大包面单右下角无字母标识，带电大包面单右下角带字母"B"标识。

**系统操作步骤：**

1.  第一步，点击"合并（consolidation）"菜单，进入扫描界面，确认出库策略信息完全一致，开始扫描，扫描成功，将小包裹与大包放置一处
    
2.  第二步，封包，点击"合并（consolidate）"，完成"确认（confirm）"，打印出大包标签。仓根据标签分类打托（举例：大包面单显示B，有B字样的大包打托在一起）
    

**注意：** 大包标签打印完毕，RMS才会展示组包完成时间。仅支持APP端操作。

**异常处理：**

*   ①**重复扫描：** 检查是否出现多个包裹同单号，同一包裹则跳过，不同包裹说明面单粘贴错误需拆包检查
    
*   ②**扫描失败：** 小包和大包的出库策略不同，检查小包是否放入错误大包，及时取出切换正确包号重新扫描
    
*   ③**普电混合报错(different package type)：** 检查小包属性是否放入正确，错误则根据明细打标取出后重新组包
    

### 2.6.3 已组包列表

如果你想要检查已组包的情况，点击扫描页面右上角列表，进入已组包列表。

**注意：** 进入大包详情页面，请确保点击的是单据卡片的空白信息，因为很容易点到LP和运单号。

### 2.6.4 组包特殊流程

1.  **大包面单打印和补打：** 在APP端确认组包结果后，需要前往PC端的大包详情页进行大包面单打印
    
2.  **缓存：** 如果你想要暂停当前大包的组包流程，你可以离开页面，之前的组包数据会被留存
    
3.  **在已组包大包中删除或新增小包：** 组包结果未提交前，可返回修改；组包结果提交后，不支持修改

*   **CN**：退国内
    
*   **HK**：退香港

### 2.7.2 特殊流程

2.  **扫描失败：** 你可以再试试或者给仓运营上报问题
    

### 2.7.3 出库特殊流程

**1. 超期未自提（Self-Pickup Overtime）：**

**逻辑：** 如果大包被组包且策略是等商家自提，但是它7天未被自提（当前时间-组包时间>7天），大包会自动被系统转成超时未自提销毁状态。

**查看：** 你可以在台账的异常页签中查看自提超时订单。

**2. 韩国要求（Korea requirements）：**

如果大包中的小包无称重重量，大包无法出库。需要拆包，补称未称重小包，无需重打大包。

**补称重流程：** 拆包 → 补称未称重小包 → 无需重打大包 → 重新组包出库。

## 2.10 硬件安装（Hardware installation）

### 2.10.1 称重配置（Weighing Configuration）

打开WET配置界面，进入称重配置界面，选择当前接入的电子秤型号跟串口。

频率默认使用**9600**，具体看电子秤的驱动程序配置，不同的电子秤会有差异。配置完成后，可以进入实操页面，在网页控制面板中查看串口消息。

**常见问题：**

1.  **找不到对应电子秤型号：** 选"万能模式"
    
2.  **COM port找不到对应的串口：**
    
    *   先在本地系统中确认电子秤对应的串口是正常打开的
        
    *   可以通过"Windows串口助手（Multifunctional Serial Port Helper）"进行串口及电子秤链接状态检查。确认电子秤数据能正常通过串口发送给当前PC电脑
        
    *   重启WET后重启浏览器
        
3.  **网页中没有正常展示称重数值，但是在网页控制面板的network pannel中查看WS消息，有对应的数据：** 通过"万能模式"链接的新型号电子秤，数据协议格式没有在当前解析库中，需要提需求给RMS技术进行新的型号解析接入。
    

**注意事项：**

1.  如果RMS页面长时间打开但没有使用，或电脑进入了休眠模式后重新进去，此时WET会与浏览器断开链接，需要刷新RMS页面重新建立链接。
    
2.  某些软件会屏蔽WS协议或端口，导致WET无法与浏览器建立WS链接。
    
3.  使用Chrome浏览器，版本号大于96。
    

### 2.10.2 打印配置（Print Configuration）

**Step 1:** 下载安装WET：`https://page.cainiao.com/cn/RfDownLoad/index.html\`

**Step 2:** 打开WET后，配置打印机

*   **条码打印机名称对应：** rmsbarcode
    
*   **面单打印机名称对应：** rmswaybill
    

如果当前电脑只链接了一台打印机，那这个名称可以随便取，因为名称匹配不正确的时候，打印任务会发送给系统默认的打印机。

RMS界面点击打印按钮后，可以在WET看到打印状态。

# 三、退货入库

本章涵盖退货入库环节的两条主要入口：上门揽收和消费者自行寄回，以及货物到达仓库后的入库作业流程。

## 3.1 上门揽业务

本地 POP、海外托管，不包含跨境业务，退平台仓业务，提供上门揽收物流服务。

### 3.1.1 上门揽收流程图

![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq76JVZ5EynAKe/img/293261ed-e482-459f-be32-fbaa0d2dc15a.png)

**图示说明（上门揽收流程）：** 退货物流单创建成功 → 用户预约 → 判断预约是否成功：预约失败则用户重新改约；用户也可主动改约。若用户退货取消，物流无需处理（CP 不允许取消的，需走"品定制特殊逻辑同步取消预约"）。预约成功 → 快递员取件 → 判断是否取件：若取件成功 → 制作是否成功：成功则快递员取件 → 门口回传 ASCAN → DSCAN；制作失败则用户改约或用户不退货。若预约取消，则用户退货取消（对 CP 退货不取消，快递员不上门）。未取件时前台转自寄。

### 3.1.2 操作规范

**用户备货：** 上门揽交接环节，快递员打印面单，用户出示 qrcode 退货；CP 通过 API 回传；

**退货包装：** 退货有包装即可，检测包装是否封包完整；

**上门揽地址要求：** 精确到门牌号，**门牌号**规则 10 位数字内，CP 需做强校验；

**上门前联系方式：** 服务商通过电话和邮箱联系用户；

**CP 关键轨迹应满足回传要求：**

| **服务段落** | **服务起始点** | **关键服务节点** | **异常态** |
| --- | --- | --- | --- |
| 揽收段 | Ascan-质检完成 | • 接单/预约成功<br>• Ascan | • 揽收失败节点<br>• 揽收失败原因<br>• 一揽失败 |
| 配送段 | Ascan-Dscan | • Ascan<br>• Dscan | • 丢失<br>• 破损<br>• 签收异常 |

### 3.1.3 特殊场景作业规范

*   1、德国：预约上门服务的订单，若消费者主动送到邮局
    
    *   服务商：若预约未取消，快递员会继续上门；邮局也会正常接收该包裹
        
    *   物流费：收取上门费、配送费
        
    *   轨迹：CP消息通知“揽收失败”（非轨迹回传，上门揽收模式无揽收失败节点），邮局在面单模式收货后回传ASCAN
        
*   2、GLS、DHL  上门揽，标准面单模式配送服务；
    
*   3、预约成功和失败VS退货取消逻辑

### 4.4.2 拍照要求说明

*   **数量要求**：包含退回包裹面单，外包装，销售包装和内物整体照3~5张，有破损、二次拆封或者异常时，对应部位另加拍1~2张照片
    
*   **拍照内容要求**：整体上，所有照片清晰完整，无遮挡，光线明亮（像素要求至少800\*600）
    
    *   **物流包装**：表面如有穿透性破损、划痕或损伤痕迹，棱角变形、箱面凹凸，重点体现
        
    *   **销售包装**：表面如有穿透性破损、划痕或损伤痕迹，棱角变形、箱面凹凸，重点体现（如退货为手机，需清楚拍摄手机盒上展示的IMEI信息）
        
    *   **内物商品**：
        
        *   包裹内商品全部取出，照片涵盖所有商品，无遮挡
            
        *   单件或少件商品完整体现销售包装盒与商品照片（如有包装盒），最小内包装如塑封销售，则不需要进行拆开拍照，仅拍摄完整外包装
            
        *   如内物有如下异常，请按要求单独补充照片
            
        *   **如退货为手机，需清楚拍摄手机盒或机身上展示的IMEI信息；（Iphone需拍摄易碎贴的情况）**

## 4.6 质检案例更新

以下为历史质检争议案例及会议判定结论，作为质检标准的补充参考。

### 4.6.1 26年1月案例

#### 4.6.1.1 汽摩配类

**1.1 汽摩配类商品配件缺失，仓库判残-配件缺失**

会议结论：GIFT非必需退回品，可以缺失；若GIFT缺失，质检判良品（通用规则，不仅限于汽摩配类）

举例单号：AP00788858159452、AP00787292987598

#### 4.6.1.2 假发类

**1.1 假发无法呈现图片效果，仓库无法判断是否属于同一顶；判残**

会议结论：

1.  判断标准仓库可从外观进行判断（如弯曲度、颜色、长度等），如有不一致判残；商品图片上的效果暂不考虑
    
2.  质检时手感发潮/使用过痕迹，质检判残
    

举例单号：AP00788697660224

**1.2 消费者购买2件商品，预报下发2件，质检图展示3件，实际收到2件，仓库无法判断真实pcs数，以及易捆内的具体数量**

会议结论：

1.  待交易下发SKU描述作为质检判断辅证，产品能力未上线前，以下发质检图为准，判残
    
2.  若一捆内的具体数量无法判断：仓库通过标题辅助判断是否包含具体数量；若标题包含数量，按照描述判断良残；若标题未清楚说明，体感无明显不足/不一致，可判良品
    

举例单号：AP00790565030947

#### 4.6.1.3 游戏机相关

**1.1 游戏机类产品，质检图显示有配件（说明书、卡包、USB等），仓库收到实物发现配件不缺失，判残**

会议结论：非GIFT的配件缺失，质检判残，次品原因"破损"damage

举例单号：AP00789451573794、AP00789259991861

**1.2 商品质检图是游戏机，仓库收到实物为手柄壳。无法判断实物商品应该是壳还是机器；判残**

会议结论：仓库依据退货商品包装、商品名称综合判断；若信息不一致，质检判残，次品原因"外观不符"

举例单号：AP00791473272845

#### 4.6.1.4 前台下发图片非真实商品照片

商品质检图为文字或者空，无对比图，仓库无法质检，判残。

## 5.2 3.5PL退国内方案

### 5.2.1 业务背景与概念

目前退国内订单中可原关退（阳关）比例较低，有可能涉及海关查验后影响到平台。基于此需要将退国内改为3.5PL模式（同时建立阳光关），保证平台合规。

**3.5PL模式核心概念：** 退国内流程分为两段：

*   **4PL段（海外仓→香港仓）：** 维持平台物流模式，由平台结算
    
*   **3PL段（香港仓→国内商家仓）：** 由商家与物流服务商直接签约，商家通过ipay平台直接与3PL服务商结算，平台不参与对账及结算
    

### 5.2.2 适用场景

*   **解决方案：** 跨境标准逆向消退\_退国内（C2G\_STANDARD\_RETURN\_CHINA）
    
*   **业务类型：** 跨境POP、跨境半托管
    
*   **国家范围：** 全量Free Return开通退国内国家（西班牙、德国、法国、英国、意大利、波兰、荷兰、美国、墨西哥、澳大利亚、韩国、沙特）
    

### 5.2.3 协议关系

*   **退海外仓+退香港服务：** 继续维持平台物流模式，由商家→平台签约
    
*   **退国内解决方案：** 海外→香港仓段落继续由商家→平台签约，维持平台物流模式；香港仓→国内段落（包含香港至国内仓运输、清关、国内派送）由商家→供应商签约
    
*   **商家签约方式：** 物流服务商入驻服务市场（限制自由下线），商家与物流服务商在AE.com进行协议签署，签约统一合同，同时需签约【商家&平台协议】及【商家&物流商协议】
    

**三份协议内容：**

*   **商家&平台协议：** 当前4PL模式协议已包含全链路服务范围，需修改退国内服务协议，明确平台只负责海外至香港仓段落+授权平台向服务商下发退国内指令及数据
    
*   **商家&物流商协议：** 增加香港仓→国内商家仓段落的3PL协议，包含服务内容、价格及赔付规则
    
*   **平台&物流商协议：** 针对退国内香港仓→国内商家仓段新增协议，包含数据交互规则、保密条款、货物交接规则
    

### 5.2.4 数据交互与仓内操作变化

**数据流核心改动点：**

*   退国内出库单依然维持一个出库AP，物流履行单拆分为4PL段落和3PL段落
    
*   3PL段起点为"3PL服务商收货成功"，终点为商家签收
    
*   3PL段需回传前台3PL订单号，所有物流轨迹展示在3PL单号，不得出现在4PL单号上

### 5.2.8 停服机制

## 5.3 二段退商家仓

### 5.3.1 业务背景与概念

目前退商家仓模式存在以下问题：S1 17CD达成率仅85%（低于目标90%）；签收争议严重（Q3商家反馈签收争议订单1.1K单，涉及交易金额10.4W美金）；退商家仓模式依赖商家确认收货后方可退款，严重影响消费者体验；末端快递为三方支付的C2C业务，CP可选范围受限。

**二段退商家仓模式：** 将退商家仓拆分为入库段和出库段：

## 5.4 销毁作业标准SOP

### 5.4.1 目的

规范海外仓滞销、残次、过期或违规商品的销毁流程，确保符合当地法律法规，防止商品二次流通，降低企业风险，维护品牌声誉。

### 5.4.2 执行范围

针对免质检销毁、质检次品出库销毁、本地仓自提未提的超时销毁等需销毁商品。

### 5.4.3 本地仓逆向销毁操作流程

![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/a/Z0qwV7PYCgOeDxkl/62fffc6603c14e5f8ae5e935b8e31b2d1580.png)

**图示说明（本地仓逆向销毁操作流程）：**

**三种销毁类型：**

1.  **免质检销毁：** 入库扫描 → 系统判断为免质检销毁 → 直接进入敏感信息脱敏处理流程
    
2.  **质检次品出库销毁：** 正常质检 → 判断是否残品：若为良品则正常出库；若为残品则生成销毁LP → 进入敏感信息脱敏处理流程
    
3.  **本地自提未提超时销毁：** 本地自提，商家预约 → 正常组包暂存 → 判断超时预约/超时自提：若商家按时自提则自提完成；若超时未提则进入敏感信息脱敏处理流程
    

**销毁执行流程（三条路径汇合）：** 敏感信息脱敏处理（参照敏感信息脱敏处理要求）→ 待销毁商品单独暂存 → 委托第三方实物销毁（需第三方资质审核、服务内容确认）→ 商品信息核实确认（敏感信息处理）→ 商品交接（附销毁交接单）→ 第三方商品销毁执行（提供销毁照片/视频、销毁证书）→ 销毁结果回传/归档（生成销毁报告）

### 5.4.4 本地仓逆向销毁类型

#### 5.4.4.1 免质检销毁

**定义：** 逆向销毁包裹经过入库扫描后，系统根据判断策略识别为"销毁（destroy）"的包裹。

*   仓内扫描录入后，按不同策略（质检、销毁、取消、无主件）进行分堆处理，送至对应的操作台
    
*   销毁（destroy）包裹，将面单上的敏感信息脱敏后，放置销毁件区域单独暂存；免质检销毁商品无需质检数量也无需质检质量（默认展示预报数量，走销毁策略无影响）
    

**系统操作（仓内）：** 打开RMS APP收货(Inbound)模块 → 扫描包裹物流单号（仅支持国际运单号） → 扫描成功后系统识别为\*\*D(Destroy)\*\*销毁类型 → 将包裹面单敏感信息脱敏后放入销毁件区域。收货扫描有四种结果：Q(质检)、C(取消)、D(销毁)、A(异常/无主件)。

#### 5.4.4.2 质检次品出库销毁

**定义：** 仓内质检后将照商品的残/良信息提交系统（确认质检结果），系统根据质检结果生成质检策略为"DESTORY NOW"（本地销毁）、"Destroy X Days Later"（在X天后销毁）的商品，同时此部分商品会系统自动生成销毁LP。

*   根据系统提示识别出待销毁商品，将面单上的敏感信息脱敏后，放置销毁件区域单独暂存
    

**系统操作（仓内）：** 入库质检 → 残/良品判断 → 残品（不含自提）→ 生成销毁LP。质检环节该界面可以直接浏览SKU对应的出库策略。当前出库策略共计**7种**：DESTORY NOW（本地销毁）、Destroy X Days Later（在X天后销毁）、Self-Pickup（本地自提）、CN（退国内）、HK（退香港）**、质检后二售、清货计划—自提**。**注：商品良残判定涉及到出库策略变化，请结合凭据谨慎判断。**

**质检策略说明：**

*   Destroy Now 为立即本地销毁
    
*   Destroy X Days Later 适用于全托场景大包存储，全托商品需要以大包形式在库存储26天（美国12天），存储到期后按大包完成销毁
    

#### 5.4.4.3 本地自提未提超时销毁

**定义：** 针对正常组包暂存超存储时效，或需自提出库但商家超时预约/商家超时自提的包裹，如超时未到仓提出，默认视为弃货，系统订单自动转为"超时销毁"状态（台账展示），仓库执行实物销毁。

*   本地自提超时：依据业务要求，将自提/在库等待货物做好库内大包存储，如出现以下场景则：
    
    *   **商家超时：** 大包组包完成后，商家未在17CD内完成预约+提货，17CD后自动转销毁
        
    *   如果因非商家原因销毁影响赔付（按货值），计入丢失场景做赔付

### 5.4.5 销毁操作注意要点

#### 5.4.5.1 待销毁商品暂存管理

*   **待销毁商品核对：** 仓库人员根据清单核对商品信息（SKU、批次、数量、状态），确保与系统记录一致
    
*   **单独隔离管理：** 将待销毁商品移至独立暂存区，贴"待销毁"标签，禁止与其他库存混放
    
*   **暂存区区分：**
    
    *   DESTROY NOW（立即销毁）和DESTROY LATER（N天后销毁）订单进行物理暂存区区隔，分开存放，并做好物理标识
        
    *   自提订单放置在自提货架/区域，超时未提订单经过扫描识别后，需转销毁的包裹，再存放至DESTROY NOW（立即销毁）的区域
        

#### 5.4.5.2 敏感信息脱敏处理

*   面单上的敏感数据提前脱敏处理，敏感信息包含且不限于消费者姓名、地址、电话、邮箱、订单号等
    
*   外包装等数据安全：对于含品牌标识等商品，销毁时需确保标识被彻底去除，防止品牌侵权
    
*   脱敏方式：可使用黑色记号笔将相关敏感信息涂黑、面单/标识撕毁/粉碎、换单覆盖，确认敏感信息在销毁后无法通过光照、物理或化学手段复原
    

**图示说明（敏感信息脱敏处理）：**

**脱敏要求：** 面单上的敏感数据需提前脱敏处理，敏感信息包含且不限于消费者姓名、地址、电话、邮箱、订单号等。外包装等数据安全：对于含品牌标识等商品，销毁时需确保标识被彻底去除，防止品牌侵权。

**脱敏方式：** 可使用黑色记号笔将相关敏感信息涂黑、面单/标识撕毁/粉碎、换单覆盖，确认敏感信息在销毁后无法通过光照、物理或化学手段复原。

**示例国家：** 西班牙、德国、法国、美国、沙特阿拉伯、韩国的面单脱敏前后对比（现状 vs 脱敏后）。

**抽检机制：** 脱敏信息处理应建立单仓抽检机制：单仓应具备内部抽检登记表，登记表有明确的by日抽检数量、稽核人员、稽核结果。

#### 5.4.5.3 销毁服务商选择

*   **资质验证：** 委托有资质的第三方处理机构（仓内委托第三方），优先选择已签约、具备资质的本地销毁服务商（如Shred-it、Stericycle、本地环保处理公司等）
    
*   **确认服务商可提供的服务：** 上门取件或现场销毁服务、带时间戳/GPS的销毁视频或照片、正式销毁证书/回执（Certificate of Destruction）等
    

#### 5.4.5.4 销毁执行

*   仓库与服务商预约取件时间
    
*   双方现场清点商品数量并签字确认《销毁交接单》，仓内二次检查商品敏感信息是否已预处理
    
    *   《销毁交接单》格式不限，根据各国实际情况填写，填写内容需包含：销毁清单（包裹单号、数量/重量）、销毁方式、执行时间、执行人、监督人的等信息
        

**图示说明（销毁交接单示例）：** 图示为韩文版（소량폐기물 처리확인증）和西班牙文版（NOTA DE RECOGIDA DE DESTRUCCIÓN DE MERCANCÍA）的销毁交接单示例。

*   服务商按约定方式进行销毁（粉碎、焚烧、化学分解等），并提供销毁凭证
    
    *   拍摄销毁前后对比视频/照片（含时间戳、商品细节）——照片需包含装运照片，卸车照片、销毁处理的现场照片
        
    *   销毁后按约定提供出具《销毁回执》（格式不限，根据各国实际情况填写，需含包裹单号/重量/数量、方式、时间、地点及负责人签字确认等信息）
        

#### 5.4.5.5 销毁凭证留存

*   **销毁记录归档：** 销毁交接单、销毁证明报告/回执、销毁现场照片/销毁过程视频/监控等记录，跟随销毁证书统一上传归档
    
*   **销毁凭证留存：** 所有销毁记录（交接单、销毁照片、销毁回执）至少保存3–7年（依当地法规要求处理）
    

#### 5.4.5.6 销毁结果回传

*   针对免质检销毁、质检次品出库销毁、自提未提的超时销毁等需销毁商品，在系统产生销毁完成节点后，对实物进行周期性（根据各地实际情况定期销毁，最长不得超过1个月）销毁，并回传销毁报告
    
*   **回传报告内容/格式：** 报告内容需包含《销毁清单》、《销毁交接单》、销毁照片/视频、《销毁证书》（可选）
    
*   **销毁结果回传链路：** 以邮件形式发送给对应国家的产品部+逆向消退业务+质控同学
    
*   **结果回传时效要求：** 本月20号前邮件提交上月销毁报告
    

#### 5.4.5.7 稽核与异常上报机制

*   **随机抽检：** 产品部应月度对销毁过程进行随机质量检查，by国家抽检样本量by月不低于总销毁量的1.25%
    
*   **异常上报：** 发现异常情况的48h内，应立即停止并上报至产品部接口人
    
*   **SOP迭代：** 当业务规则发生迭代，逆向解决方案应与逆向质控进行拉通，质控对对销毁流程评估和迭代
    
*   **总部质控稽核机制：** 总部质控稽核结果如不达标，对应产品部及单仓应对存量异常进行处理，并由产品部输出整改报告

# 六、出库与配送

商品处置完成后，进入出库与配送环节。本章涵盖小包打包、大包装箱（组包）、出库及台账管理。详细的系统操作步骤请参考 **二、RMS系统操作指南** 对应章节。

## 6.1 小包打包

### 6.1.1 小包打包作业流程

根据不同的装箱策略和带电标识，将对应SKU及数量打为小包，（完成称重）完成包裹封口，贴上对应的面单。

**打包的方式：**

*   将需"退国内/退香港/本地自提/本地销毁"SKU单品做复核，装入PE袋/箱子打包为小包，粘贴"退国内/退香港/本地自提/本地销毁"LP面单
    
*   出库面单覆盖入库面单
    

**注意：**

1.  黑码商品合并到本地销毁策略中，完成销毁即可
    
2.  同一个策略下，带电和不带电的包裹分别打包
    

### 6.1.2 小包打包系统操作

1.  **打包：** 整理SKU及对应数量，出库策略显示为"Self-Pickup"(本地自提)、"CN"（退国内）、"HK"（退香港）、"DESTORY NOW"（本地销毁）、"Destroy After 'yyyy-mm-dd'"（在'yyyy-mm-dd'日期销毁）**、质检后二售、清货计划—自提**，共计**7种**，按左侧展示该策略下合计的SKU种类数和总物品数量，图片右下角展示该SKU应包含的数量，需核实清晰，准确装入对应的商品和数量
    
2.  **称重（可选）：** 装入完成后，可以进行小包裹称重（直接由硬件设备采集结果），不支持直接输入重量数据
    
3.  **打印面单：** 完成内物核对后，点击打印运单(Print Waybill)，将包裹进行封口，将面单进行粘贴。（支持批量打印/重复打印，但需注意准确粘贴，包裹打包完成后产生的异常不可逆）
    

## 6.2 大包装箱（组包）

**支持端：只支持APP**

### 6.2.1 组包作业流程

1.  **大包绑定：** 将小包对应出库策略一致的大包，进行逐票扫描，系统绑定。逐票扫描，装大包（系统扫描完成，即为大小包映射关系绑定）
    
    *   **注意：** 不同流向小包不能混装；普电不能混装
        
2.  **自提线上化组包操作：** 仓库人员定期查看系统中预约待组包的提醒，操作流程如下：
    
    *   组包扫描小包条码，如果商家已预约则会显示预约标识，反之则无预约标识
        
    *   已预约的小包组同一个大包，未预约的小包组同一个大包，二者不可也不支持混包
        
    *   同一个商家，不同预约日期的自提小包，不支持组同一个大包
        
    *   组包过程中如遇某个小包因丢失/破损等原因无法组包，需删除对应小包后按照预约单继续组大包
        
    *   "已预约"组包过程中，如果商家取消，则整个预约单被取消，阻断当前组包操作
        
    *   "已预约"组包完成后，在PC端点击预约单打印、发货资料打印（服务商自提场景下），并把面单粘贴在大包上，预约单等自提交接时签字使用
        
3.  **大包存储：** 依据业务要求，补充标识（如要求口岸清晰（退国内/退香港）、数量标记等），将自提/在库等待货物做好库内大包存储
    
    *   等待至商家预约时间进行提货；如果商家一直未提货，等待系统销毁指令
        
    *   退国内/退香港存储到交干日期前
        
    *   全托商品需要以大包形式在库存储26天（美国12天），存储到期后按大包完成销毁
        

**注意：** 部分仓有消退&不可达两种业务，请将消退的包裹和不可达的包裹分开提供清单，实物包裹也分开打包，箱子上清晰明显标注消退和不可达标签。短期不可达不适用RMS系统。

### 6.2.2 组包系统操作

**作业流程：** 大包装箱时，需在小包封口后粘贴唯一面的大包面单。面单需覆盖原入库面单，避免出库运输时扫描到入库面单信息。同一出库策略下，带电与不带电SKU须分开打包。普通大包面单右下角无字母标识，带电大包面单右下角带字母"B"标识。

**系统操作步骤：**

1.  第一步，点击"合并（consolidation）"菜单，进入扫描界面，确认出库策略信息完全一致，开始扫描，扫描成功，将小包裹与大包放置一处
    
2.  第二步，封包，点击"合并（consolidate）"，完成"确认（confirm）"，打印出大包标签。仓根据标签分类打托（举例：大包面单显示B，有B字样的大包打托在一起）
    

**注意：** 大包标签打印完毕，RMS才会展示组包完成时间。仅支持APP端操作。

**异常处理：**

*   ①**重复扫描：** 检查是否出现多个包裹同单号，同一包裹则跳过，不同包裹说明面单粘贴错误需拆包检查
    
*   ②**扫描失败：** 小包和大包的出库策略不同，检查小包是否放入错误大包，及时取出切换正确包号重新扫描
    
*   ③**普电混合报错(different package type)：** 检查小包属性是否放入正确，错误则根据明细打标取出后重新组包
    

### 6.2.3 已组包列表

如果你想要检查已组包的情况，点击扫描页面右上角列表，进入已组包列表。

**注意：** 进入大包详情页面，请确保点击的是单据卡片的空白信息，因为很容易点到LP和运单号。

### 6.2.4 组包特殊流程

1.  **大包面单打印和补打：** 在APP端确认组包结果后，需要前往PC端的大包详情页进行大包面单打印
    
2.  **缓存：** 如果你想要暂停当前大包的组包流程，你可以离开页面，之前的组包数据会被留存
    
3.  **在已组包大包中删除或新增小包：** 组包结果未提交前，可返回修改；组包结果提交后，不支持修改
    

## 6.3 出库（Outbound）

**支持端：只支持APP**

### 6.3.1 出库流程

扫描大包面单，系统显示出库结果：

*   **SP**：自提
    
*   **CN**：退国内
    
*   **HK**：退香港
    

### 6.3.2 特殊流程

1.  **重复扫描：** 意味着大包已经做过出库操作
    
2.  **扫描失败：** 你可以再试试或者给仓运营上报问题
    

### 6.3.3 出库特殊流程

**1. 超期未自提（Self-Pickup Overtime）：**

**逻辑：** 如果大包被组包且策略是等商家自提，但是它7天未被自提（当前时间-组包时间>7天），大包会自动被系统转成超时未自提销毁状态。

**查看：** 你可以在台账的异常页签中查看自提超时订单。

**2. 韩国要求（Korea requirements）：**

如果大包中的小包无称重重量，大包无法出库。需要拆包，补称未称重小包，无需重打大包。

**补称重流程：** 拆包 → 补称未称重小包 → 无需重打大包 → 重新组包出库。

# 七、异常处理方案

## 7.1 异常类型定义

### 7.1.1 多面单

**定义：** 一个包裹上包含两个及以上数量且为不同单号的面单，包裹内物混装复杂，无法一对一完整质检上架。

**处理流程：** 发现多面单包裹，先**登记为异常件**，再完成入仓扫描，完成可在当日完成标准件作业后按如下质检流程单独处理：

1.  入库扫描，查验多面单所有对应单号的入库方案：
    
    *   打开异常登记菜单，扫描任一一个面单，登记异常件，异常原因选择"多面单（multiple labels）"
        
    *   打开入仓扫描菜单，将包裹逐个面单进行扫描，完成入仓节点
        
2.  开箱：将包裹（含部分需质检）进行开箱，需拆包平铺所有实际到货实物
    
3.  人工目视&质检，对比退回商品是否和系统记录的信息一致（外观/SKU/数量）：参照包裹面对应的多个订单的系统图片，核对和匹配商品图片和实物是否存在一致：
    
    *   若商品SKU与订单预报SKU信息对应，按商品和订单对应分离，分开进行品类数量下一步验证，按实际情况入相应sku良品或残品
        
    *   若商品SKU与订单预报SKU信息部分对应，将已经匹配上的商品实物与其他实物进行分离，对订单及实物进行下一步验证：
        
        *   当商品SKU量少于订单SKU，订单中对应缺失的SKU-选择"丢失（LOST）"选项并且填写预报数量
            
        *   当商品SKU量多于订单SKU，多余商品无预报的添加为黑码商品
            
    *   若商品SKU与订单信息均无法匹配，在一个订单列表中添加黑码商品及对应数量，其他订单中质检界面选择"丢失"的次品原因，登记预报数量即可
        

**举例说明：** 假设一个包裹面单3张（LP1/2/3）订单对应商品为a/b/c，假设实际商品有多件、少件品类不对应的情况，其对应入库方式如下：

| **面单** | **LP1** | **LP2** | **LP3** | **货物质检结果&处理方式** |
| --- | --- | --- | --- | --- |
| **预报sku** | **a** | **b** | **c** |  |
| **收到**实际商品a\b\c | a-正品 | b-正品 | c-正品 | LP1 \LP2\LP3 良品各1个 |
| 实际商品a\b | a-正品 | b-正品 | 无 | LP1 \LP2 良 2个、LP3 丢失1个 |
| 实际商品d\e\f | d-残 | e-残 | f-残 | LP1 \LP2\LP3 : a-lost、d\e\f-黑码 |
| _以上假设质检的商品及其销售包装均完好且1个SKU1件_ |  |  |  |  |

### 7.1.2 无主件

**定义：** 仓库在处理包裹时，发现无预报、无指令、无法入库扫描（排除非菜鸟订单，除取消订单）。

**处理流程：** 判断面单情况，是否为AE面单（如有LP订单/有AE标签等）

*   **有面单且明显是AE的**（面单正常、订单存在、但系统原因设备无法扫描识别）：入仓扫描登记异常件"other-系统异常无法扫描"→履约订单重推/系统错误修复→仓一周内重新扫描操作
    
*   **有面单但不是AE**（可能是消费者使用其他物流送仓）：等待7WD，再次扫描，若仍无法识别直接完成销毁
    
*   **无面单**（可能是消费者直接送仓）：入仓扫描登记异常件"No data"，直接完成销毁
    

![image](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/a/aDE4WABnuDzvJ5Wk/731bb8362f994a0fa3035488b0accee61580.png)

**时效考核：** 因无正常订单关联，不在时效考核范围内

### 7.1.3 取消但到仓订单

**定义：** 仓库在入仓扫描处理包裹时，发现展示为"取消（CANCELLED）"。

**处理流程：** 将取消包裹单独存放，等待7天，7日内若有入库指令/通知（线下）下发，按常规件再次入库扫描完成质检及后续动作；若无指令/通知，再进行销毁处理。

**时效考核：** 取消订单因履约单已关闭，删除时效考核

### 7.1.4 重复面单

**定义：** 重复使用面单发多个包裹到仓，仓库在入仓处理包裹时发现订单之前已经处理过第一个包裹并已经是完成状态，无法继续上架。

**处理流程：** 入仓扫描登记异常件-"other-重复面单"，仓执行销毁。

### 7.1.5 空包

**定义：** 包裹打开无内物/仅含填充物/价值极低，与订单商品信息差异大，无法完成质检上架。

**处理流程：** 在质检环节

*   仍在系统中上传3张图片：1)上传到包裹检查的是包裹图 2)上传到拆包检查的是货物图 3)上传到商品描述的也是货物图
    
*   质检数量：根据预报的SKU数量，选择"丢失（lost）"选项并且填写预报数量，实物进行销毁
